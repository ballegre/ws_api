GroverWebServices Module¶

GroverWebServices is a python module defining class objects that model and provide access to PerfAnt performance testing results stored in Grover databases.

The module provides two basic functional categories. The first, is Low Level Grover Web Services API that directly models the Grover web services interface. The second is a value added service (this section) implementing high level abstractions to make accessing results more convenient.

class GroverWebServices.GroverCommandLineArgument(runarg, peak_default=True)[source]¶

Object class representing Grover run results at a specified load point. runarg is a string that takes the form runid[:lp_id|lp_index], where runid is Grover runid, lp_id is optional Grover loadpoint identifier (of the form iter_loadpoint, eg. 1_10000) or lp_index is optional numerical index (start=1). Special forms of symbolic load point specification are one of: "all", "peak" or "last". If the optional load point is omitted then peak is assumed unless peak_default=False.

Note:SFS is a notable exception to the standard lp_id format.

Example:

import GroverWebServices
run_iter = GroverWebServices.GroverCommandLineArgument("120330gav:peak")
run_iter = GroverWebServices.GroverCommandLineArgument("120330gav:5_22000")
run_iter = GroverWebServices.GroverCommandLineArgument("120330gav:5")

All the above create the same run object instance at the same load point.

Object instantiation does little more than syntax checking. Explicit invocation of additional methods are required to visit the Grover database and retrieve test parameters, results, and command output.

class GroverWebServices.GroverResult[source]¶

An object class representing a grover run result identifed by runid, eg.:

result = GroverResult('112233abc')

GroverResult objects contain test parameters and overall test results for a performance test run under PerfAnt. The GroverResult object is an abstraction containing globally significant test information such as settings, overrides, load points and, lists of commands collected at each load point.

class GroverWebServices.GroverEventSearch[source]¶

Object class respresenting Grover eventSearch web service call. Event search parameters are extensive and must be checked at time of execution. Object instantiation simply provides a place holder for adding member attributes that are converted to XML name/value pairs in HTML request. The search executes when the search() method is invoked. To see full list of parameters display contents of GroverEventSearch._service.parameters attribute.

Attribute holding multi-dimensional dictionary indexed by event id as primary key. Within event id dictionary is two indexes: 'contexts' and 'runids'. GroverStatsDisk Module¶

GroverStatsDisk is a python module defining classes and objects relevant for the “stats disk” command.

class GroverStatsDisk.GroverStatsDisk(run)[source]¶

Object class representing “stats disk” command.

Parameters:run – The GroverWebServices.GroverCommandLineArgument object representing the runid and load point where command is retrieved.

Example:

import GroverWebServices
import GroverStatsDisk

run = GroverWebServices.GroverCommandLineArgument("120330gav:peak")
stats_disk = GroverStatsDisk.GroverStatsDisk(run)
print stats_disk.output()
